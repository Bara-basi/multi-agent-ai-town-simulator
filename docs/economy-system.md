# 经济系统设计

## 总览

这个项目的经济系统不是简单的“随机涨跌”，而是围绕 **均值回归 + 对数噪声 + 库存恢复 + 交易摩擦** 四层结构搭建的。目标不是做一个绝对真实的市场，而是让它同时满足：

- 有明显波动，可以形成交易机会
- 不会长期失控，价格始终围绕基本面
- 不同品类有不同风险结构
- 生存品和投机品拥有不同玩法定位

## 1. 价格模型：均值回归 + 对数噪声

市场价格更新写在 `MarketComponent.generate_price()` 中：

```python
lnP = lnP + kappa * (lnbase - lnP) + rng.normal(0.0, sigma, size=lnP.shape)
next_price = exp(lnP)
```

其数学形式可以写成：

$$
\log P_{t+1} = \log P_t + \kappa (\log P^{*} - \log P_t) + \varepsilon_t
$$

$$
\varepsilon_t \sim \mathcal{N}(0, \sigma^2)
$$

其中：

- $P_t$：当前价格
- $P^{*}$：基础价格 `base_price`
- $\kappa$：回归速度
- $\sigma$：波动强度

这个模型本质上是“定义在对数价格上的离散均值回归过程”。

## 2. 为什么在对数空间建模

在对数空间建模有三个直接优势：

- 价格天然大于 0，不会出现负价格
- 控制的是相对波动率，而不是绝对价差
- 低价商品和高价商品可以共享同一套波动逻辑

所以这里的噪声是乘法意义上的，而不是简单的线性加减。

## 3. KAPPA：为什么取 0.11

当前配置中：

```python
KAPPA = {
    "comsumable": 0.11,
    "valuable": 0.11,
}
```

$\kappa = 0.11$ 表示每天回收约 11% 的价格偏离。若忽略噪声，则偏离量按：

$$
(1 - \kappa)^t = 0.89^t
$$

递减。

偏离半衰期约为：

$$
t_{\frac{1}{2}} \approx \frac{\ln 0.5}{\ln 0.89} \approx 6 \text{ 天}
$$

这意味着市场既有趋势窗口，又不会让价格长期飘离基本面。

两类商品共用同一个 `KAPPA`，是因为这里希望把“品类差异”主要放在波动率，而不是放在回归速度。

## 4. SIGMA：为什么消耗品和贵重品差异很大

当前配置：

```python
SIGMA = {
    "comsumable": 0.15,
    "valuable": 0.4,
}
```

### 消耗品：$\sigma = 0.15$

对应单日典型相对波动大约：

$$
e^{0.15} \approx 1.16
$$

即约 $\pm 16\%$ 的量级。  
这让水、面包、肉有足够的价格弹性，但不会破坏生存节奏。

### 贵重品：$\sigma = 0.4$

对应单日典型相对波动大约：

$$
e^{0.4} \approx 1.49
$$

即接近 $\pm 50\%$ 的量级。  
这让银戒指、黄金更像高波动资产而不是普通商品。

因此，`SIGMA` 实际上定义了市场中的风险分层：

- 消耗品偏稳健
- 贵重品偏投机

## 5. 长期稳定性：为什么价格不会炸掉

这个模型稳定，不是因为“随机数小”，而是因为均值回归过程本身存在稳态。

对数价格围绕 $\log P^{*}$ 的平稳方差近似为：

$$
\mathrm{Var}(\log P) \approx \frac{\sigma^2}{1 - (1 - \kappa)^2}
$$

在 $\kappa = 0.11$ 时：

$$
1 - 0.89^2 = 0.2079
$$

这意味着：

- 消耗品长期波动相对温和
- 贵重品长期波动显著更大
- 两者都会被均值回归项拉回有限区间

所以市场会持续给出机会，但不会随着回合数增长而完全失真。

## 6. 库存恢复：价格之外的第二层约束

市场每天都会补货，但不是一次补满：

```python
self._stock[item_id] = min(default_quantity, current_stock + DEFAULT_MARKET_STOCK_INCREASE)
```

当前设置：

- 默认库存 `DEFAULT_MARKET_STOCK = 40`
- 每日补货 `DEFAULT_MARKET_STOCK_INCREASE = 10`

它的作用是：

- 玩家扫货会留下短期痕迹
- 市场不会被一次行为永久掏空
- 供给恢复是缓慢的，不是瞬时复原

这让市场兼具扰动性和恢复性。

## 7. 买卖价差：交易摩擦

如果价格有波动但没有摩擦，策略很容易退化成机械套利。因此项目里给不同商品设置了 `sellRatio`：

- 水、面包：`0.85`
- 肉：`0.8`
- 贵重品：接近或等于 `1`

含义很明确：

- 消耗品更偏“使用型商品”
- 贵重品更偏“投机型资产”

`sellRatio` 与 `SIGMA` 共同决定了某个品类到底更适合生存消费，还是更适合财富博弈。

## 8. 设计结果：两条不同的生存路径

这套参数最终推导出了两种很自然的玩法：

- 低风险路径：围绕消耗品做补给管理，优先保证生存
- 高风险路径：围绕贵重品做波段交易，追求更高收益

这不是写死在文本里的规则，而是由参数自然生成的市场结构。

## 9. 为什么这套系统足够有效

从工程角度看，这套市场模型参数很少，但语义很清晰：

- `base_price`：基本面锚点
- `KAPPA`：回归速度
- `SIGMA`：波动强度
- `sellRatio`：交易摩擦
- `stock + restock`：供给约束

这足以构成一个可解释、可调参、可长期运行的小型经济系统，也足以支撑一个作品集级别的多智能体交易场景。
