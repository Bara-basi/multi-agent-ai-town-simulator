# AITown 决策层项目概述

## 1. 项目定位
这是一个 AI 小镇（AITown）项目的“决策层”原型代码，职责是：
- 组织世界模型（角色、地点、物品、市场等）。
- 为每个 NPC 构建“观察 -> 规划 -> 行动 -> 反思”的循环。
- 通过大模型（OpenAI Chat Completions）驱动 NPC 的生存与交易行为。

当前仓库更接近“可运行的早期框架/实验版”，核心模块已经拆分，但存在较多实现不一致和待完善点。

## 2. 目录与模块结构
- `main.py`：程序入口，初始化世界、执行器、每个角色的运行时循环。
- `runtime/`
  - `load_data.py`：从 CSV 读取 `actor/item/location` 定义并组装 `Catalog`。
  - `build_state.py`：把定义层转换为运行时状态（`ActorState`、`LocationState`）。
  - `runtime.py`：`AgentRuntime`，负责 tick 调度（plan/act/reflect 与 action retry）。
- `model/`
  - `definitions/`：定义层数据结构（`ActorDef`、`ItemDef`、`LocationDef`、`Catalog`、`Inventory`、`LLM` 等）。
  - `state/`：运行时状态（`WorldState`、`ActorState`、`LocationState`、`ActionResult`）。
  - `brains/`：智能体脑模块（`AgentBrain`、`PromptBuilder`、`EffectManager`）。
- `actions/`
  - `action_registry.py`：动作注册中心（handler + validators + alias）。
  - `handlers.py`：内置动作处理逻辑（`move/consume/buy/sell/sleep/wait/finish/skill`）。
  - `validators.py`：动作合法性校验（地点、库存、金额、物品持有）。
  - `executor.py`：动作标准化 + 校验 + 执行封装。
- `data/`：世界静态数据（角色、物品、地点 CSV）。
- `debug_log/prompt/`：提示词与结构化包日志（md/json），便于调试 LLM 决策链路。
- `config/config.py`：全局常量与运行时参数配置（含 `AgentRuntimeConfig`）。

## 3. 运行主流程
1. `main.py` 读取 CSV -> 构建 `Catalog`。
2. 基于 `Catalog` 构建 `ActorState` / `LocationState`，再组装 `WorldState`。
3. 为每个 actor 创建：
   - `Agent`（封装 LLM + PromptBuilder）
   - `AgentRuntime`（单 actor 的决策循环）
4. 每个 actor 以异步循环持续 tick：
   - `observe`：从 `WorldState` 生成快照
   - `plan`：按步数间隔触发全局计划
   - `act`：让 LLM 产出动作 JSON
   - `execute`：通过 `ActionExecutor` 执行动作
   - `reflect`：按间隔触发总结反思

## 4. 世界模型（核心对象）
- `Catalog`：静态定义索引（items/locations/actors），并提供 snapshot 给提示词使用。
- `WorldState`：全局动态状态容器，提供 `observe(actor_id)` 输出统一观察视图。
- `ActorState`：角色状态（位置、金额、属性 attrs、背包、可达地点、记忆、状态标记）。
- `LocationState`：地点动态组件容器，当前目标是支持 Market 组件。
- `Inventory`：物品数量管理（has/add/remove/snapshot）。

## 5. 智能体决策链路
- `Agent` 对外提供：
  - `plan(obs)`：生成阶段计划文本。
  - `act(obs)`：生成下一步动作（JSON）。
  - `reflect(obs)`：生成反思总结文本。
- `PromptBuilder` 负责：
  - 构造统一 PromptPacket（分 section：背景、规则、状态、任务、动作指南等）。
  - 合成市场商品信息、地点信息、近期事件等上下文。
  - 将 prompt 同时落盘到 `debug_log/prompt/*.md|*.json`。
- `LLM`（`OpenAIModel.py`）调用 OpenAI Chat Completions；`restrict='json'` 时要求 JSON object 输出并反序列化。

## 6. 动作执行系统
- `ActionExecutor` 将模型输出动作标准化为统一视图（支持 dict/对象两种输入）。
- 通过 `action_registry` 查找动作入口：
  - 先跑 validators（失败直接返回 `ActionResult`）。
  - 再执行 handler（异常转为 `CRASH`）。
- 已注册动作包含生存与交易基础能力：
  - 位置/流程：`move`、`wait`、`finish`、`sleep`
  - 资源：`consume`
  - 交易：`buy`、`sell`
  - 技能：`skill`（含 `skill-*` 兼容入口）

## 7. 数据与配置
- 静态数据来源：
  - `data/actor.csv`
  - `data/item.csv`
  - `data/location.csv`
- 运行配置（`AgentRuntimeConfig`）：
  - 生存危险阈值：hunger/thirst/fatigue
  - `max_action_retries`
  - `max_replan_after_action_error`
  - 循环保护阈值（重复动作 / 重复交易）
  - `reflect_min_interval_steps`

## 8. 当前实现特征（现状描述）
从代码现状看，这个决策层已经具备以下雏形：
- 多角色并发 tick 运行框架。
- 统一观察快照与提示词构建链路。
- 动作注册/校验/执行的可扩展机制。
- 市场交易与生存动作的基础接口。
- Prompt 调试日志沉淀能力。

## 9. 一句话总结
该仓库是一个面向“AI NPC 生存+贸易”场景的决策层原型：主线是以 `WorldState` 为环境、以 `AgentRuntime` 为循环、以 LLM + PromptBuilder 为决策脑、以 ActionExecutor 为执行器，当前已形成端到端骨架并进入稳定性与一致性打磨阶段。
